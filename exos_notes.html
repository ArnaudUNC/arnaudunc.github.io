Exercice 1 ✔️

  • Date début: vendredi 20 août 2021
  • Date limite pour rendre: jeudi 26 août 2021 à 13h30
  • A rendre sur Moodle: soumettre uniquement votre unique fichier .py, dans l'onglet évaluations du cours d'algo. et prog. 2 en Python.
  • Retard possible: non.

C'est un exercice à faire SEUL(E), la triche sera activement recherchée et sanctionnée par un zéro à toutes les copies identiques même si seuls certains morceaux de code sont identiques.

Vous allez créer un programme qui permet à l'utilisateur de manipuler une liste depuis la console: il/elle pourra y ajouter des éléments.

Voici un exemple de à quoi doit ressembler votre programme lors de son execution:

Votre programme sera noté sur les critères suivants lors de son exécution:

  1. (1 point) Le programme se répète indéfiniment tant que l'utilisateur n'a pas entré la lettre "q"
  2. (1 point) Le programme saute une ligne (donc une ligne vide) dans la console entre chaque répétition pour une meilleure lisibilité
  3. (1 point) Affichage de la liste à chaque répétition
  4. (1 point) Affichage des choix possibles de l'utilisateur à chaque répétition
  5. (1 point) La lettre "a" permet d'ajouter un élément de type str par défaut dans la liste
  6. (2 point) Si l'élément entré par l'utilisateur représente un nombre entier, le nouvel élément doit être de type int automatiquement
  7. (2 point) Si l'élément entré par l'utilisateur représente un nombre à virgule, le nouvel élément doit être de type float automatiquement
  8. (1 point) Le programme affiche un message de "fin" avant de quitter 

				liste = []
				réponse_utilisateur = ""
				while réponse_utilisateur != "q":
					print("Votre liste:")
					print(liste)
					print("Actions possibles:")
					print("a - ajout d'un nouvel élément dans la liste")
					print("q - quitter")
					réponse_utilisateur = input("Votre choix: ")
					if "a" == réponse_utilisateur:
						# ↓ On récupère le texte de l'utilisateur sous forme de str,
						# mais peut-être qu'il s'agit d'un nombre entier ou d'un nombre à virgule.
						nouvel_élément = input("Entrez nouvel élément: ")
						# ↓ D'abord on tente de convertir le texte entré par l'utilisateur en int.
						try:
							nouvel_élément = int(nouvel_élément)
						except:
							# ↓ Si on est ici, c'est que la conversion en int a échoué,
							# on tente donc maintenant la conversion en float.
							try:
								nouvel_élément = float(nouvel_élément)
							except:
								# ↓ Si la conversion en float échoue à son tour, ça signifie que
								# que le texte de l'utilisateur est juste du texte générique,
								# donc on ne fait rien, car il est déjà sous forme de str.
								# On est obligé de mettre ce block "except" car le block "try" précédent
								# DOIT être suivi de son bloc "except" (ou un bloc "finally", voir CM4).
								# Mais rappelez-vous: un bloc de code en Python ne peut PAS être vide, il doit avoir
								# au moins une ligne de code, on utilise donc "pass" qui ne fait rien de spécial.
								pass
						# ↓ Après une conversion éventuelle en int ou float, on ajoute l'élément dans la liste.
						liste.append(nouvel_élément)
					print()
				print("Merci et à bientôt!")

Exercice 2

  • Date début: dimanche 19 septembre 2021
  • Date limite pour rendre: jeudi 23 septembre 2021 à 13h30
  • A rendre sur Moodle: soumettre uniquement votre unique fichier .py, dans l'onglet évaluations du cours d'algo. et prog. 2 en Python.
  • Retard possible: non.

C'est un exercice à faire SEUL(E), la triche sera activement recherchée et sanctionnée par un zéro à toutes les copies identiques même si seuls certains morceaux de code sont identiques.

Vous devez créer 7 fonctions, le tout dans UN SEUL module (un seul fichier .py). Voici ce qui doit être strictement respecté comme demandé dans les consignes (sinon les points ne seront pas comptés):

  • le nom que vous donnez à chaque fonction doit être EXACTEMENT celui demandé, au caractère prêt: majuscule/minuscule, accents ou pas.
  • le nombre et l'ordre des paramètres pour chaque fonction doivent être EXACTEMENT comme demandés
  • la valeur de retour de chaque fonction doit respecter ce qui est demandé
  • en règle générale tenez-vous en uniquement à ce qui est demandé, ni plus ni moins. Vous serez noté sur le respect scrupuleux du "contrat" de chaque fonction (nom, params, actions, et valeur de retour)
  • aucun code en dehors des fonctions n'est demandé

Et voici là où vous êtes libres de faire comme vous voulez:

  • le nom du module (fichier .py) n'importe pas
  • l'ordre des fonctions n'importe pas
  • le nom de chaque paramètre des fonctions n'importe pas
  • comment la fonction fait ce qu'elle doit n'importe pas, tant qu'elle le fait correctement

Les fonctions sont demandées dans aucun ordre particulier, donc faites-les dans l'ordre que vous préférez.

Fonction 1

Créer une fonction nommée construire_chaine, qui prend un seul paramètre de type list qui contient des objets de n'importe quel type. La fonction doit mettre bout à bout la représentation sous forme de str de chaque objet contenu dans la liste, placer un (et un seul) caractère espace entre chaque, et renvoyer la chaîne finale comme résultat.

La représentation sous forme de str doit être celle par défaut de l'objet (dont vous avez l'habitude jusqu'ici), par exemple le int 8 donnera '8', et la liste [10, "coucou", True] donnera "[10, 'coucou', True]", donc les deux mis bout à bout donneront la chaîne "8 [10, 'coucou', True]"

Fonction 2

Créer une fonction nommée avoir_operation_ensembles, qui prend 3 paramètres:

  1. un ensemble, donc de type set
  2. un autre ensemble, donc de type set également
  3. un str

Le 3eme paramètre doit donner une opération à réaliser sur les deux ensembles donnés en paramètres 1 et 2, ses valeurs acceptées sont:

  • "intersection"
  • "différence"
  • "union"

La fonction retourne:

  • le int -1 si les paramètres 1 et/ou 2 ne sont pas des ensembles
  • le int -2 si l'opération demandée n'est pas une de celles acceptées
  • le résultat (de type set) de l'opération demandée sur les deux ensembles, si les paramètres sont valides. Pour la différence, il est important que ça soit le 1er ensemble moins le second, l'ordre est important.

Fonction 3

Créer une fonction nommée ajouter_entier_non_nul_dans_liste, qui prend 2 paramètres:

  1. une liste
  2. un élément qui peut être de n'importe quel type

La fonction doit ajouter l'élément à la fin de la liste SEULEMENT SI celui-ci est un int différent de zéro. S'il est d'un autre type de donnée, ou s'il vaut zéro, l'élément ne doit PAS être ajouté dans la liste.

La fonction retourne:

  • le int 1 si l'ajout a été fait
  • le int -1 dans le cas contraire

Fonction 4

Créer une fonction nommée ajouter_entiers_non_nuls_dans_liste, qui prend 2 paramètres:

  1. une liste
  2. une seconde liste

La fonction doit ajouter à la fin de la premiere liste, tous les éléments (et dans le même ordre) de la seconde liste qui sont des int différents de zéro. Les éléments qui n'obéissent pas à ces critères sont simplement ignorés et donc non ajoutés à la première liste.

La fonction retourne:

  • le bool True si TOUS les éléments de la seconde liste ont été ajoutés
  • le bool False si au moins un élément n'a PAS été ajouté

Fonction 5

Créer une fonction nommée supprimer_element_de_liste, qui prend 2 paramètres:

  1. une liste
  2. un élément qui peut être de n'importe quel type

La fonction doit supprimer l'élément s'il se trouve dans la liste. Si l'élément est présent plusieurs fois dans la liste, la suppression doit enlever uniquement celui de plus petit indice, c'est à dire celui qui apparaît en premier dans la liste. La liste sera donc plus courte de un élément après la suppression.

La fonction retourne:

  • le int 1 si la suppression a été faite
  • le int -1 si la suppression n'a pas été faite, c'est à dire quand l'élément n'est pas présent dans liste

Fonction 6

Créer une fonction nommée supprimer_indice_de_liste, qui prend 2 paramètres:

  1. une liste
  2. un indice, de type int

La fonction doit supprimer l'indice spécifié de la liste. La liste sera donc plus courte de un élément après la suppression.

La fonction retourne:

  • le int 1 si la suppression a été faite
  • le int -1 si l'indice spécifié n'existe pas dans la liste
  • le int -2 si l'indice spécifié n'est pas un int

Fonction 7

Créer une fonction nommée creer_liste, qui prend 2 paramètres:

  1. un int, censé représenter une borne inférieure
  2. un second int, censé représenter une borne supérieure

La fonction doit créer une liste qui contient les nombres entiers (int) entre la borne inférieure inclue, et la borne supérieure exclue, en comptant de 1 en 1.

La fonction retourne:

  • le int -1 si au moins un des deux paramètres n'est pas un int
  • le int -2 si la borne inférieure est égale ou plus grande que la borne supérieure
  • la liste résultat si les bornes sont correctes
