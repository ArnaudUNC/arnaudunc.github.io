Variation 1

1) Donnez le nom exact des 7 types de donnée fondamentaux qu'on connait en Python, et pour chacun préciser s'ils sont mutables ou immutables.

int, immutable; float, immutable; str, immutable; tuple, immutable; list, mutable; bool, immutable; NoneType, immutable.

2) Quelle fonction permet de connaître l'identité d'un objet? Quel est le type de son résultat.

id(); int

3) Quelle fonction permet de connaître le type d'un objet? Quel est le type de son résultat.

type(); type

4) Soit le code suivant:

L1 = ["a", "b", "c"]
L2 = ["a", "b", "c"]

Donner les résultats des expressions suivantes:

L1 == L2
L1 is L2

et expliquer pourquoi on a ces résultats.

True pour "==" car les deux objets de type list ont la même valeur, mais False pour "is" car ce sont deux objets différents, ils n'ont donc pas la même identité.

5) Pour chaque ligne ci-dessous, dire si c'est une expression ou pas, et pourquoi ça l'est ou pas. Quand c'est une expression donner sa valeur finale et son type final.

10
ma_variable = 3
ma_variable
ma_variable + 5
return ma_variable + 1
ma_variable > 3
ma_variable - 1 == 3
max([5+6, 10])

10 est une expression, car c'est une constante, donc un objet dont la valeur est écrite littéralement. La valeur de l'expression est le int 10.

ma_variable = 3 n'est pas une expression, car c'est une instruction.

ma_variable est une expression, car c'est une variable, la valeur de l'expression est celle de l'objet référencé par la variable, qui est le int 3.

ma_variable + 5 est une expression, résultat de l'opérateur + entre deux int, la valeur est le int 8.

return ma_variable + 1 n'est pas une expression. ma_variable + 1 en est une, mais avec return devant l'ensemble devient une instruction.

ma_variable > 3 est une expression, résultat de l'opérateur > entre deux int, la valeur est le bool False.

ma_variable - 1 == 3 est une expression. D'abord l'opérateur - est calulé entre deux int car il a une priorité supérieure à celle de l'opérateur ==. Puis l'opérateur == est évalué entre deux int. La valeur finale de l'expression est le bool False.

max([5+6, 10]) est une expression, résultat de l'appel de la fonction max, avec comme paramètre la liste [11,10], le calcul de 5+6 se faisant avant de pouvoir passer la liste en paramètre. Le résultat final de l'expression est le int 11.

6) Expliquer ce qu'est la variable __name__: ce qu'elle "contient" comme donnée, et dans quel(s) namespace(s) elle existe.

Ce nom existe dans chaque module, et référence un objet de type str qui est le nom du module (le nom du fichier .py) dans lequel elle existe si le module est importé, "__main__" si le module est celui qui est exécuté.

7) Expliquer ce qu'est une variable locale.

C'est une variable qui existe dans le namespace d'une fonction en cours d'exécution.

8) Le namespace des variables globales est-il unique? Pourquoi?

Non, car ce qu'on appelle le namespace global est en fait le namespace d'un module, il y a donc autant de namespaces "globaux" que de modules.

Variation 2

1) Qu'est-ce que CPython?

CPython est l'implémentation de référence de Python. Une implémentation est une version spécifique et concrète de la spécification de Python.

2) Quelles sont les 3 informations associées à tout objet en Python.

Son type, sa valeur et son identité.

3) Expliquer ce qu'est une expression, et donner au moins 3 "choses" qui peuvent la composer.

Une expression est l'écriture dans le code d'un calcul (ou d'une valeur) qui peut être très court ou très long, mais qui s'évalue au final à une seule et unique valeur, donc un seul objet, et qui a donc un type final unique. On utilise les expression à certains endroits où un calcul (ou une valeur) est nécessaire, par exemple à droite de l'affectation avec "=", la condition du bloc if, la condition de la boucle while, ou encore à droite du "in" dans la boucle for. Une expression peut être composée de valeurs littérales, de noms (variables), d'appels de fonctions, d'opérateurs, et de parenthèses pour contrôler la priorité des opérateurs.

4) Soit le code suivant:

a = 5
Sachant que l'objet 5 est immutable, est-il possile de faire ensuite:
a += 1
Expliquer votre réponse.

Oui c'est possible car l'objet 5 de type int ne change pas, c'est la variable "a" qui change, et référence un nouvel objet de type int et de valeur 6.

5) Qu'est ce que sont techniquement "is" et "==". Puis expliquer ce qu'ils font, le type de donnée de leur résultat, et leur différence.

Ce sont deux opérateurs binaires, et booléens car ils ont tous deux un bool comme résultat. "is" compare l'identité des deux objets à gauche et à droite, et vaut vrai si les deux identités sont égales, faux sinon. "==" compare la valeur des deux objets à gauche et à droite, et vaut vrai si les deux objets ont la même valeur, faux sinon.

6) Soit les deux modules A et B suivants:

Module A:
print(__name__)

Module B:
import A
print(__name__)
Si on exécute le module B, que verra-t-on affiché dans la console?

On verra:
A (à cause du print dans le module A, qui affiche le nom du module, puisque celui-ci est importé)
__main__ (à cause du print dans le module B, qui affiche "__main__" puisque ce module est celui qui est exécuté)

7) Expliquer ce qu'est une variable globale

C'est une variable qui existe dans le namespace d'un module.

8) Le namespace d'un module est-il le même que le namespace global? Pourquoi?

Oui, tout simplement car les deux noms (namespace global et namespace d'un module) sont synonymes. Ce qui implique qu'il n'y a pas de namespace global unique, mais un namespace par module.

10) Si on essaye de faire fonctionner le code suivant que verra-t-on affiché dans la console, et expliquer pourquoi.

x = 10
try:
	x = 50
	x = float(str(1/0) + "toto")
	x = 100
except ValueError:
	print("Erreur ValueError", x)
except ZeroDivisionError:
	print("Erreur ZeroDivisionError", x)
except:
	print("Erreur générique", x)
print("Bonjour")

On verra affiché:
Erreur ZeroDivisionError 50
Bonjour

Parce que l'opération 1/0 devra être calculée en premier sur la ligne où elle se situe, et cela provoquera une ZeroDivisionError. La ligne ne pouvant se terminer correctement, x référence toujours l'objet précédent, soit le int 50. La ligne x = 100 ne sera pas exécutée car elle est dans le try après la ligne qui provoque l'exception.

Le Bonjour c'est parce que ce print est à la suite de la structure try-except, il s'exécute donc qu'il y ait eu une erreur ou non.
